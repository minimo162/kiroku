# Kiroku ユーザーマニュアル

## 1. このマニュアルの対象

このマニュアルは、経理担当者が Kiroku を使って業務記録を取得し、CSV を Copilot Excel で分析するまでの手順をまとめたものです。

対象読者:

- Kiroku を初めて使う実務担当者
- チーム展開を支援する管理者
- 配布済みアプリの運用手順を確認したい IT 部門

## 2. インストール

前提:

- Windows 11
- WebView2 利用可能
- ローカル保存先に書き込み権限がある

手順:

1. 配布された `.msi` を実行します。
2. 画面の案内に従ってインストールを完了します。
3. スタートメニューから `Kiroku` を起動します。

補足:

- 初回起動時はセットアップウィザードが開きます。
- `llama-server` はインストーラーに同梱されている想定です。

## 3. 初回セットアップ

ウィザードの流れ:

1. `ようこそ` 画面で用途を確認します。
2. モデルダウンロードを開始します。
3. ダウンロード完了後、テストキャプチャを実行します。
4. 完了画面でダッシュボードへ進みます。

確認ポイント:

- ダウンロード中に進捗率が更新される
- テストキャプチャでエラーが出ない
- ダッシュボードに遷移できる

## 4. 日常操作

### 起動と常駐

- アプリを閉じてもシステムトレイに常駐します。
- トレイメニューからウィンドウ再表示、記録開始、記録停止、終了ができます。

### 記録開始と停止

- ダッシュボードの `記録開始` ボタンを押します。
- 停止するときは `記録停止` を押します。
- ホットキー `Ctrl+Shift+R` でも切り替えできます。

### ダッシュボードで見る項目

- 本日のキャプチャ枚数
- 有効フレーム数
- VLM 処理済み件数
- 次回バッチ予定時刻
- 最新の記録一覧

## 5. 設定

`設定` 画面で次を変更できます。

- キャプチャ間隔
- 差分閾値
- 夜間バッチ有効化
- バッチ開始時刻
- VLM ホスト
- VLM 最大トークン数
- 保存先ディレクトリ
- VLM プロンプト
- マスキングルール

### VLM プロンプトの調整

- `システムプロンプト`: 経理業務向けの記述方針を指定します。
- `ユーザープロンプト`: 出力の粒度や文体を指定します。

推奨:

- まず既定値のまま運用する
- 修正が多いアプリだけ追加ルールを入れる
- 業務用語は画面に根拠がある場合のみ増やす

### マスキングルール

用途例:

- 取引先名を `[取引先]` に置換
- 金額を `[金額]` に置換
- 個人名を `[担当者]` に置換

設定例:

|対象|パターン|置換後|正規表現|
|---|---|---|---|
|取引先名|株式会社A|[取引先]|いいえ|
|金額|`\b\d{3},\d{3}\b`|[金額]|はい|

## 6. 履歴検索と記述確認

### 履歴

`履歴` 画面では以下で絞り込みできます。

- キーワード
- アプリ
- 開始日
- 終了日

検索結果をクリックすると、該当日の `記述プレビュー` に移動します。

### 記述プレビュー

`記述プレビュー` では次を実行できます。

- 日付ごとの記録確認
- 画像の確認
- VLM 記述の手動修正
- 変更履歴の確認

## 7. CSV エクスポート

手順:

1. `エクスポート` 画面を開きます。
2. 日付範囲を指定します。
3. 必要に応じて対象アプリを選びます。
4. `VLM 処理済みのみ` を必要に応じて切り替えます。
5. `マスキングを適用` を確認します。
6. `CSV をエクスポート` を押します。
7. 保存後、必要なら Explorer でファイルを開きます。

出力列:

- `timestamp`
- `app`
- `window_title`
- `description`

## 8. Copilot Excel への入力手順

1. エクスポートした CSV を Excel で開きます。
2. Copilot を開きます。
3. 次のプロンプトを貼り付けます。

```text
このExcelには、業務PCの画面状態を10秒間隔で記録した時系列データがあります。
各行は:
- timestamp: 記録時刻
- app: アクティブアプリケーションのプロセス名
- window_title: ウィンドウタイトル
- description: AIが画面内容を記述したテキスト

以下を新しいシートに作成してください:

■シート「手順書」
連続する記述から一連の操作フローを識別し、業務ごとに番号付き手順として記述。
各手順に[開始時刻][終了時刻][使用アプリ]を付与。
同じ業務が複数回出現する場合はパターンを統合。

■シート「業務一覧」
業務名 / 業務カテゴリ / 実施回数 / 平均所要時間 / 使用アプリケーション / 備考

■シート「時間分析」
業務カテゴリ別の時間配分集計、時間帯別の業務パターン、
改善余地がありそうな作業へのコメント。
```

## 9. よくあるトラブル

### VLM 接続テストに失敗する

- `設定` 画面で `VLM ホスト` が `127.0.0.1:8080` などローカルアドレスになっているか確認
- `llama-server` が起動しているか確認
- セキュリティソフトがローカル通信を妨げていないか確認

### キャプチャが増えない

- 記録状態が `停止中` になっていないか確認
- キャプチャ間隔が長すぎないか確認
- 画面差分閾値が高すぎないか確認

### CSV に機密情報が残る

- `設定` 画面でマスキングルールを追加
- `エクスポート` 画面で `マスキングを適用` を有効化

## 10. アンインストール

1. 記録中なら停止します。
2. `設定` 画面で必要な CSV を先に出力します。
3. Windows の `アプリ > インストールされているアプリ` から `Kiroku` をアンインストールします。

注意:

- 退避していないローカルデータは削除される可能性があります。

## 11. IT 部門向け補足

配布時の確認項目:

- `.msi` の配布元が信頼済みであること
- WebView2 が利用可能であること
- モデル保存先に十分な空き容量があること

ローカルビルド時の補足:

- `src-tauri/binaries/llama-server-x86_64-pc-windows-msvc.exe` を配置してから `npm run tauri:build` を実行します。
- GitHub Actions では `LLAMA_SERVER_SIDECAR_URL` と `LLAMA_SERVER_SIDECAR_SHA256` を設定すると sidecar を自動取得できます。
- MSI のコード署名を有効にする場合は、GitHub Secrets に `AZURE_TENANT_ID` `AZURE_CLIENT_ID` `AZURE_CLIENT_SECRET`、GitHub Variables に `TRUSTED_SIGNING_ENDPOINT` `TRUSTED_SIGNING_ACCOUNT_NAME` `TRUSTED_SIGNING_CERTIFICATE_PROFILE_NAME` を設定します。
